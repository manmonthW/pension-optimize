## 📱 软件简介

### 这是什么软件？

"养老最优解"是一款专业的养老金测算小程序，帮助您科学规划退休时间，找到最划算的退休方案。

### 能帮我做什么？

**核心价值：**
- ✅ **模拟未来缴费**：自动计算从现在到退休的每一笔缴费
- ✅ **对比多种方案**：比较不同退休年龄的养老金差异
- ✅ **智能推荐最优解**：基于科学算法推荐最佳退休时间
- ✅ **回本周期分析**：告诉您多久能收回缴费成本

### 为什么需要这个软件？

**解决的痛点：**
1. 不知道延迟退休是否划算
2. 不清楚不同缴费档次的实际收益
3. 无法评估4050补贴、失业金对养老金的影响
4. 想知道自己未来能领多少养老金

---

## 🎯 核心功能

### 1. 未来缴费模拟

**功能说明：**
从您当前的年龄开始，逐月模拟到退休日期的所有缴费情况。

**考虑因素：**
- 当前就业状态（在职/灵活就业/失业等）
- 失业金领取情况（最长24个月）
- 4050补贴抵扣（每月669.58元）
- 个人账户年度计息（4%复利）
- 缴费档次调整（60%/100%/200%/300%）

**计算内容：**
- 每月个人实际支付金额
- 每月进入个人账户的金额
- 累计缴费年限增长
- 个人账户余额变化

---

### 2. 多方案对比

**功能说明：**
根据您的人员类型，自动生成3-5个退休方案进行对比。

**方案类型：**

| 人员类型 | 正常退休 | 延迟退休方案 |
|---------|---------|-------------|
| 男职工 | 60岁 | 61岁、62岁、63岁 |
| 女干部 | 55岁 | 56岁、57岁、58岁 |
| 女工人 | 50岁 | 51岁、52岁、53岁、55岁 |

**对比维度：**
- 月养老金金额
- 总缴费金额
- 回本周期
- 个人账户余额
- 缴费年限

---

### 3. 智能推荐算法

**功能说明：**
采用三维度加权评分，科学评估每个方案的综合价值。

**评分维度：**
1. **月养老金增长率（50%）** - 延迟退休能多领多少钱
2. **回本周期优化率（30%）** - 能否更快收回缴费成本
3. **缴费成本节约率（20%）** - 总缴费是否减少

**推荐原则：**
综合评分最高的方案为推荐方案（不一定是养老金最高的方案）。

---

### 4. 详细数据展示

**提供信息：**
- ✅ 基础养老金（统筹部分）
- ✅ 个人账户养老金
- ✅ 月养老金总额
- ✅ 总缴费金额
- ✅ 退休时账户余额
- ✅ 回本周期（月数）
- ✅ 累计缴费年限

点击任意方案卡片，查看该方案的详细信息。

## 🧮 计算逻辑详解

### 一、未来缴费模拟

**逐月计算规则：**

#### 1. 在职状态
```
每月个人缴费 = 缴费基数 × 8%
进入个人账户 = 缴费基数 × 8%
```

**示例：**
- 缴费基数：10000元
- 个人每月支付：10000 × 8% = 800元
- 全部进入个人账户：800元

#### 2. 灵活就业状态
```
每月个人缴费 = 缴费基数 × 20%
进入个人账户 = 缴费基数 × 8%
```

**示例：**
- 缴费基数：10000元
- 个人每月支付：10000 × 20% = 2000元
- 进入个人账户：10000 × 8% = 800元
- 剩余12%（1200元）进入统筹账户

**重要说明：**
灵活就业人员需要自己承担全部20%的费用，但只有8%进入个人账户，剩余12%用于基础养老金统筹。

#### 3. 失业补贴内
```
每月个人缴费 = 0元
进入个人账户 = 0元
可领取失业金 = 2286元/月（北京标准）
```

**说明：**
- 失业金期间不缴社保
- 最长领取24个月
- 不增加缴费年限
- 不增加个人账户余额

#### 4. 失业补贴外
```
每月个人缴费 = 0元
进入个人账户 = 0元
```

**说明：**
失业金已领完，暂时不缴费。

#### 5. 4050补贴抵扣

**适用人群：**
- 女性满40岁
- 男性满50岁
- 就业困难人员

**抵扣规则：**
```
实际支付 = 原缴费金额 - 669.58元
```

**示例：**
- 灵活就业，缴费基数10000元
- 原缴费：10000 × 20% = 2000元
- 享受4050补贴后：2000 - 669.58 = 1330.42元

---

### 二、养老金计算公式

#### 1. 基础养老金（统筹部分）

**公式：**
```
基础养老金 = 退休金核算基数 × 1% × 缴费年限
```

**北京2025年参数：**
- 退休金核算基数：12049元

**示例：**
- 缴费年限：25年
- 基础养老金 = 12049 × 1% × 25 = 3012.25元/月

**说明：**
- 基础养老金来自统筹账户
- 缴费年限越长，基础养老金越高
- 与个人缴费档次无关，只看年限

#### 2. 个人账户养老金

**公式：**
```
个人账户养老金 = 退休时个人账户余额 ÷ 计发月数
```

**计发月数表（N表）：**
| 退休年龄 | 计发月数 |
|---------|---------|
| 50岁 | 195月 |
| 55岁 | 139月 |
| 60岁 | 93月 |

**示例：**
- 50岁退休
- 个人账户余额：156000元
- 计发月数：195月
- 个人账户养老金 = 156000 ÷ 195 = 800元/月

**说明：**
- 退休年龄越大，计发月数越少
- 计发月数越少，每月领的越多
- 账户余额越多，每月领的越多

#### 3. 月养老金总额

**公式：**
```
月养老金 = 基础养老金 + 个人账户养老金
```

**完整示例（女工人，50岁退休）：**
```
缴费年限：25年
个人账户：156000元
退休年龄：50岁

基础养老金 = 12049 × 1% × 25 = 3012.25元
个人账户养老金 = 156000 ÷ 195 = 800元
月养老金总额 = 3012.25 + 800 = 3812.25元
```

---

### 三、个人账户计息规则

**年度复利计息：**
```
每年年底：账户余额 = 账户余额 × (1 + 4%)
```

**长期影响示例：**
- 初始余额：10万元
- 10年后：10万 × 1.04^10 = 14.8万元
- 增长：4.8万元

**说明：**
个人账户每年都会计息，时间越长，复利效应越明显。

---

### 四、回本周期计算

**公式：**
```
回本周期（月） = 总缴费金额 ÷ 月养老金
```

**示例：**
- 总缴费：30万元
- 月养老金：5000元
- 回本周期 = 300000 ÷ 5000 = 60个月（5年）

**意义：**
- 回本周期越短越好
- 表示多快能收回缴费成本
- 之后领的都是净收益

## 🏆 评分标准说明

### 综合评分公式

```
总分 = 养老金增长率 × 0.5 + 回本优化率 × 0.3 + 成本节约率 × 0.2
```

### 三大评分维度

#### 1. 月养老金增长率（权重50%）

**计算方法：**
```
增长率 = (推荐方案养老金 - 正常退休养老金) ÷ 正常退休养老金
```

**示例：**
- 正常退休（50岁）：3500元/月
- 延迟1年（51岁）：4000元/月
- 增长率 = (4000 - 3500) ÷ 3500 = 14.3%

**贡献分：**
- 增长率 × 5（最高50分）

**解释：**
延迟退休最大的好处是增加养老金，这是最重要的指标。

---

#### 2. 回本周期优化率（权重30%）

**计算方法：**
```
优化率 = (正常退休回本月数 - 推荐方案回本月数) ÷ 正常退休回本月数
```

**示例：**
- 正常退休回本：60个月
- 延迟1年回本：50个月
- 优化率 = (60 - 50) ÷ 60 = 16.7%

**贡献分：**
- 优化率 × 3（最高30分）

**解释：**
回本越快，说明投资效益越高，降低长寿风险。

---

#### 3. 缴费成本节约率（权重20%）

**计算方法：**
```
节约率 = (正常退休总缴费 - 推荐方案总缴费) ÷ 正常退休总缴费
```

**示例：**
- 正常退休总缴费：30万元
- 延迟1年总缴费：28万元
- 节约率 = (30 - 28) ÷ 30 = 6.7%

**贡献分：**
- 节约率 × 2（最高20分）

**解释：**
虽然延迟退休多缴1年，但因为提前不缴的年份更多，总缴费可能反而减少。

---

### 评分示例

**完整案例：**
```
正常退休（50岁）：
- 月养老金：3500元
- 回本周期：60个月
- 总缴费：30万元

推荐方案（51岁）：
- 月养老金：4000元
- 回本周期：50个月
- 总缴费：28万元

评分计算：
1. 养老金增长率 = 14.3% → 得分 14.3 × 5 = 71.5分（超过50分，取50分）
2. 回本优化率 = 16.7% → 得分 16.7 × 3 = 50.1分（超过30分，取30分）
3. 成本节约率 = 6.7% → 得分 6.7 × 2 = 13.4分

总分 = 50 + 30 + 13.4 = 93.4分
```

## 📝 输入参数说明

### 1. 人员类型

**男职工：**
- 正常退休年龄：60岁
- 可延迟至：61、62、63岁

**女干部：**
- 正常退休年龄：55岁

**女工人：**

---

### 2. 出生年月

**作用：**
- 计算当前年龄
- 计算退休日期
- 确定模拟期间长度

**注意：**
- 必须真实准确
- 影响所有计算结果

---

### 3. 当前缴费基数

**定义：**
每月缴纳社保的基数金额。

**北京2025年范围：**
- 最低：7162元
- 最高：35811元

**如何查询：**
1. 查看工资条
2. 登录社保APP
3. 咨询单位HR

**影响：**
- 基数越高，每月缴费越多
- 基数越高，个人账户增长越快
- 基数越高，未来养老金越高

---

### 4. 已缴费年限

**包含：**
- 实际缴费年限
- 视同缴费年限

**如何查询：**
- 社保APP查询
- 社保局查询

**重要性：**
直接影响基础养老金金额。

---

### 5. 个人账户余额

**定义：**
当前个人账户累计金额。

**如何查询：**
- 社保APP实时查询
- 社保局柜台查询

**重要性：**
直接影响个人账户养老金。

---

### 6. 缴费档次（level）

**四个档次：**

**60%档（最低）：**
- 实际基数 = 当前基数 × 60%
- 适合：收入低、经济压力大
- 特点：缴费少，养老金也少

**100%档（标准）：**
- 实际基数 = 当前基数 × 100%
- 适合：大多数人
- 特点：平衡缴费和收益

**200%档（较高）：**
- 实际基数 = 当前基数 × 200%
- 适合：收入较高，想增加养老金
- 特点：缴费多，养老金高

**300%档（最高）：**
- 实际基数 = 当前基数 × 300%
- 适合：高收入，追求高养老金
- 特点：缴费最多，养老金最高

**示例：**
```
当前缴费基数：10000元
城市最低基数：7162元
城市最高基数：35811元

60%档：10000 × 60% = 6000元 → 低于最低限，调整为7162元
100%档：10000 × 100% = 10000元
200%档：10000 × 200% = 20000元
300%档：10000 × 300% = 30000元
```

### 2. 评选标准

**三维度详解：**
- 💰 月养老金增长率（50%权重）
- ⏱️ 回本周期优化率（30%权重）
- 💸 缴费成本节约率（20%权重）

**为什么这样设计：**
- 养老金高低最重要（50%）
- 回本快慢很重要（30%）
- 成本多少次重要（20%）

---

### 3. 推荐理由

**四大优势对比：**

**▲ 月养老金提升**
- 绝对金额：比正常退休多多少钱
- 增幅百分比：增长了百分之几

**✓ 回本时间缩短**
- 提前月数：快多少个月
- 缩短比例：提前百分之几

**↓ 总缴费节省**
- 节省金额：省多少万元
- 节省比例：节省百分之几

**📈 个人账户余额**
- 增加金额：多多少元
- 增长比例：增加百分之几

**综合评分**
- 0-100分评分
- 分数越高，方案越优

---

### Q8: 回本周期什么意思？

**A:**
**定义：**
多久能领回缴费成本。

**计算：**
回本月数 = 总缴费 ÷ 月养老金

**示例：**
- 总缴费：30万元
- 月养老金：5000元
- 回本：300000 ÷ 5000 = 60个月（5年）

**意义：**
- 回本越快越好
- 回本后领的都是赚的
- 活得越长，越划算
